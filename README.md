# Rhythm C++ : High-Performance osu!mania Clone

**Rhythm C++**는 C++와 SFML 프레임워크를 기반으로 제작된 4-Key 건반형 리듬 게임 프로젝트입니다. .osu 표준 비트맵 파싱, BASS 라이브러리를 이용한 정밀한 오디오 동기화, SQLite를 활용한 데이터 관리 등 리듬 게임의 핵심 메커니즘을 직접 구현하였습니다.

## 주요 특징

### 1. 정밀한 오디오 동기화 및 저지연 환경
* **BASS 라이브러리 연동**: SFML 기본 오디오 모듈의 레이턴시 한계를 극복하기 위해 BASS 라이브러리를 채택하였으며, 버퍼 및 업데이트 주기를 최적화하여 타격음 지연을 최소화했습니다.
* **시간 기반 동기화 시스템**: 프레임워크의 갱신 속도에 의존하지 않고, 음악의 실제 재생 시간(BASS_ChannelGetPosition)을 기준으로 모든 노트의 위치 업데이트와 판정 로직을 수행하여 높은 정밀도를 확보했습니다.

### 2. .osu 표준 비트맵 파서 구현
* **데이터 파싱**: 텍스트 기반의 .osu 파일을 분석하여 [HitObjects] 섹션의 데이터를 실시간으로 읽어와 메모리에 적재합니다.
* **Mania 모드 알고리즘**: 비트맵의 x좌표를 4개의 레인으로 변환하는 수식을 적용하고, 비트 연산을 통해 단일 노트와 롱노트(Long Note)를 구분하여 처리합니다.

### 3. 시스템 최적화 및 확장성
* **효율적인 메모리 관리**: 판정이 완료된 노트를 O(1) 시간 복잡도로 제거하기 위해 std::deque 자료구조를 활용하여 장시간 플레이 시에도 안정적인 성능을 유지합니다.
* **가변 해상도 대응**: 프로젝트 내 SCALE 매크로 설정을 통해 다양한 화면 해상도 환경에서도 UI와 게임 레이아웃이 정해진 비율을 유지하도록 설계했습니다.

### 4. 데이터 영속성 및 UX
* **SQLite 데이터베이스**: SQLite3를 연동하여 사용자 ID별 최고 점수를 기록하며, 결과 화면에서 실시간 Top 10 리더보드를 출력하는 기능을 구현했습니다.
* **동적 리소스 로딩**: 선택된 곡의 .osu 파일 내부 정보를 참조하여 배경 이미지와 음악 파일을 자동으로 매칭하고 로드합니다.

## 기술 스택

* **Language**: C++17
* **Graphics/Input**: SFML 3.0
* **Audio Engine**: BASS Library
* **Database**: SQLite3
* **Build System**: CMake
* **Platform**: macOS (CoreFoundation 연동)
