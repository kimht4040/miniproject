# 🎹 C++ osu!mania Clone Project

[cite_start]C++와 SFML을 활용하여 osu!mania 형식의 리듬 게임을 개발하는 프로젝트입니다. [cite: 1]

## 🛠 1단계: 프로젝트 환경 설정 및 라이브러리 연동
* **Graphic/Input**: **SFML**을 사용하여 화면 출력 및 사용자 입력을 처리합니다.
* **Audio**: 정밀한 시간 측정 및 저지연(Low-latency) 재생을 위해 **BASS Library**를 연동합니다.
* **IDE**: **CLion** 환경에서 **CMake**를 통해 라이브러리를 링크합니다.
* **Resource**: 테스트용 `.osu` 비트맵 파일과 배경음악(`.mp3`) 파일을 준비합니다.

---

## 📂 2단계: .osu 파일 파서(Parser) 구현
텍스트 기반의 비트맵 데이터를 읽어 메모리에 저장합니다.
* **구조체 정의**: `ManiaNote` 구조체를 정의하여 레인(Lane), 시작 시간, 종료 시간, 롱노트 여부 등을 저장합니다.
* **섹션 필터링**: `std::ifstream`으로 파일을 읽어 `[HitObjects]` 섹션 이후의 데이터만 추출합니다.
* **데이터 파싱**:
    * 쉼표(`,`)를 기준으로 데이터를 분리(Tokenizing)합니다.
    * **레인 계산**: $Lane = \lfloor x \times 4 / 512 \rfloor$ 공식을 사용하여 x 좌표를 레인 번호로 변환합니다.
    * **노트 구분**: 비트 연산(`type & 128`)을 통해 단타와 롱노트를 구분하여 저장합니다.

---

## 🕒 3단계: 시간 동기화 및 노트 생성 로직
프레임이 아닌 **음악의 실제 재생 시간**을 기준으로 게임을 동기화합니다.
* **음악 재생**: BASS 라이브러리를 통해 현재 음악 재생 위치(ms)를 `currentTime`으로 추적합니다.
* **노트 위치 업데이트**:
    * `time_diff = 노트시간 - 현재시간` 공식을 사용하여 위치를 결정합니다.
    * **Scroll Speed** 배율을 적용하여 $y$ 좌표를 계산하고 판정선으로 이동시킵니다.
    * 판정 전 일정 시간(예: 1000ms)부터 화면에 렌더링을 시작합니다.

---

## 🎯 4단계: 렌더링 및 판정(Judgement) 시스템
실제 플레이어의 입력에 따른 판정 로직을 구현합니다.
* **매니아 렌더링**: 4K 레인에 맞춰 수직으로 떨어지는 노트를 렌더링합니다.
* **키 입력 처리**: `D, F, J, K` 등의 키 입력 시 가장 가까운 노트와의 시간 오차를 계산합니다.
    * **Perfect**: $\pm 50ms$ 이내
    * **Miss**: 노트를 놓치거나 판정 시간이 지났을 때
* **UI 이펙트**: 판정 결과와 콤보(Combo)를 화면에 출력합니다.

---

## 🚀 5단계: 정교화 및 확장 (선택 사항)
* **롱노트 로직**: 누르고 있는 동안의 판정 지속 및 수직 바(Bar) 형태의 렌더링을 구현합니다.
* **스코어링**: 점수 계산 공식 및 결과 화면을 추가합니다.
* **최적화**: 처리된 노트를 벡터에서 효율적으로 관리하여 성능을 개선합니다.
* **지식 관리**: **PARA 방법론**을 적용하여 Obsidian 등에 개발 로그를 관리합니다.
