## 1. 프로젝트 개요

- **프로젝트명:** (가칭) Rhythm Cpp
    
- **개발 환경:** C++, CLion, CMake (macOS)
    
- **사용 라이브러리:** SFML (그래픽, 오디오, 윈도우, 이벤트 처리)
    
- **게임 장르:** 4-Key 건반형 리듬게임 (D, F, J, K 키 사용)
    

## 2. 핵심 게임플레이 기능

### 2.1. 음악 재생 및 동기화 시스템

- **음악 재생:** `sf::Music`을 사용하여 게임 시작과 동시에 배경 음악(BGM)을 재생합니다.
    
- **시간 동기화:** 프레임이 아닌 **음악의 현재 재생 시간(`getPlayingOffset()`)**을 기준으로 게임의 모든 요소를 업데이트합니다.
    

### 2.2. 노트 생성 및 낙하 로직

- **노트 렌더링:** `sf::RectangleShape` 등을 이용해 화면 상단에서 생성된 노트를 그립니다.
    
- **이동 방식:** 정해진 '낙하 속도'와 '음악 재생 시간'을 곱하여 매 프레임 노트의 Y좌표를 계산해 아래로 떨어뜨립니다.
    
- **메모리 관리:** 화면 밖으로 완전히 벗어난(Miss 판정) 노트는 즉시 메모리에서 해제(`pop` 또는 `erase`)하여 최적화합니다.
    

### 2.3. 입력 및 판정 (Hit Detection)

- **키 입력 감지:** s, d, k, l 키의 눌림 이벤트를 감지합니다.
    
- **판정 기준:** 해당 라인에서 가장 밑에 있는 노트의 '목표 도달 시간'과 '사용자가 키를 누른 시점의 음악 시간'의 차이(오차)를 계산하여 판정합니다.
    

|**판정 등급**|**오차 범위 (예시)**|**콤보 처리**|
|---|---|---|
|**PERFECT**|± 0.05초 이내|+1|
|**GREAT**|± 0.10초 이내|+1|
|**GOOD**|± 0.15초 이내|+1|
|**MISS**|± 0.15초 초과 (또는 입력 없이 지나침)|**0으로 초기화**|

### 2.4. 점수 및 시스템

- **점수 계산:** 노트 1개당 기본 점수를 부여하고, 판정 등급(Perfect, Great 등)에 따라 가중치를 곱해 누적 점수에 더합니다.
    
- **콤보 시스템:** 연속으로 노트를 맞출 경우 콤보가 증가하며, Miss 발생 시 콤보가 0으로 초기화됩니다.
    

## 3. UI (사용자 인터페이스) 기능

- **게임 레이아웃:**
    
    - 4개의 세로 라인(Lane) 구분선.
        
    - 화면 하단에 노트가 도달해야 하는 고정된 **판정선(Judgment Line)**.
        
- **실시간 텍스트 피드백 (`sf::Text`):**
    
    - 현재 점수 표시 (우측 상단).
        
    - 현재 콤보 수 표시 (화면 중앙).
        
    - 가장 최근 노트의 판정 결과(PERFECT, MISS 등)를 짧은 시간 동안 화면에 표시.
        

## 4. 데이터 관리 (채보 시스템)

- **채보(Beatmap) 파싱:** 게임이 시작될 때 텍스트(.txt) 또는 CSV 파일을 읽어와 노트 데이터를 메모리(`std::vector` 또는 `std::queue`)에 적재합니다.
    
- **데이터 구조 예시:**
    
    - 형식: `[떨어질 라인 번호(0~3)], [판정선에 닿아야 할 시간(초)]`
        
    - 예: `0, 1.25` (첫 번째 줄에 1.25초 타이밍에 맞춰 노트가 떨어지도록 생성)
        

## 5. 개발 진행 순서 (Milestones)

- **Phase 1:** 창 띄우기, 판정선 그리기, SFML 기본 텍스처/도형 렌더링.
    
- **Phase 2:** 하드코딩된 노트 1개를 시간 흐름에 맞춰 정확하게 떨어뜨리기.
    
- **Phase 3:** 키보드 입력을 받아 오차 시간에 따른 판정(Perfect/Miss) 텍스트 띄우기.
    
- **Phase 4:** 채보 파일(txt)을 읽어와 여러 개의 노트를 연속으로 떨어뜨리기.
    
- **Phase 5:** 오디오(`sf::Music`) 연동 및 점수/콤보 UI 등 살 붙이기.
